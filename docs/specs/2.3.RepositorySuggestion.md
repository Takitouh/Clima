# 2.3 Repository Suggestion - User Feedback Management

## Overview

Implementation of the user feedback/suggestions repository layer. Handles CRUD operations for
storing user feedback messages in the local database. This repository manages user-provided
suggestions about the weather app functionality.

**Location:** `data/repository/impl/RepositorySuggestion.kt`

**Interface:** `data/repository/IRepositorySuggestion.kt`

**Status:** ✅ COMPLETE

---

## Entity: Suggestion (User Feedback)

```kotlin
@Entity(tableName = "suggestion")
data class Suggestion(
    @PrimaryKey(autoGenerate = true)
    var id: Int = 0,
    @ColumnInfo(name = "full_name")
    var fullName: String,
    var message: String,
    @ColumnInfo(name = "created_at_utc")
    var createdAtUtc: String
)
```

**Purpose:**

- Stores user-provided suggestions and feedback
- Maintains user contact information for follow-up
- Tracks when feedback was submitted
- Enables feedback analytics and improvement tracking

---

## DAO: DaoSuggestion

```kotlin
@Dao
interface DaoSuggestion {
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insert(suggestion: Suggestion): Long

   @Query("SELECT * FROM suggestion")
   suspend fun obtainAll(): List<Suggestion>

    @Update
    suspend fun update(suggestion: Suggestion): Int

   @Query("DELETE FROM suggestion WHERE id = :id")
   suspend fun delete(id: Int): Int
}
```

**Methods:**

- `insert()`: Adds new feedback (replace on conflict)
- `obtainAll()`: Retrieves all feedback records
- `update()`: Updates existing feedback
- `delete()`: Deletes feedback by ID

## Repository Interface: IRepositorySuggestion

### Method: insert()

**Creates a new feedback record in the database.**

```kotlin
suspend fun insert(suggestion: Suggestion): Long
```

**Parameters:**

- `suggestion`: Suggestion object containing full name, message, and date

**Returns:**

- `Long`: ID of newly inserted record

**Error Handling:**

- Throws `Exception` if database operation fails

**Use Cases:**

- Store new user feedback immediately after submission
- Create audit trail of user suggestions

**Example:**

```kotlin
val feedback = Suggestion(
   fullName = "John Doe",
   message = "Please add hourly weather forecast feature",
   createdAtUtc = "2026-05-01 10:30:00" // YYYY-MM-DD HH:MM:SS MUST BE IN UTC!!
)

val feedbackId = repositorySuggestion.insert(feedback)
Log.d("Feedback", "Feedback stored with ID: $feedbackId")
```

---

### Method: obtainAll()

**Retrieves all feedback records from database.**

```kotlin
suspend fun obtainAll(): List<Suggestion>
```

**Returns:**

- `List<Suggestion>`: All feedback records, empty list if none exist

**Error Handling:**

- Throws `Exception` if database operation fails

**Use Cases:**

- Display feedback history in admin panel
- Analyze user suggestions for feature prioritization
- Generate feedback reports
- Backup user feedback data

**Example:**

```kotlin
val allFeedback = repositorySuggestion.obtainAll()
Log.d("Feedback", "Retrieved ${allFeedback.size} feedback records")

allFeedback.forEach { suggestion ->
   println("${suggestion.fullName}: ${suggestion.message}")
}
```

---

### Method: update()

**Modifies an existing feedback record.**

```kotlin
suspend fun update(suggestion: Suggestion): Int
```

**Parameters:**

- `suggestion`: Suggestion object with updated data (must have ID set)

**Returns:**

- `Int`: Number of records updated (0 or 1)

**Error Handling:**

- Throws `Exception` if database operation fails

**Important:**

- Suggestion must have valid ID from database
- Returns 0 if ID doesn't exist

**Use Cases:**

- Correct mistaken feedback entries
- Mark feedback as reviewed or processed
- Update feedback status

**Example:**

```kotlin
val existingFeedback = repositorySuggestion.obtainAll().firstOrNull()
if (existingFeedback != null) {
   existingFeedback.message = "Updated feedback message"
   val updatedCount = repositorySuggestion.update(existingFeedback)
    Log.d("Feedback", "Updated $updatedCount record(s)")
}
```

---

### Method: delete()

**Removes a feedback record from database.**

```kotlin
suspend fun delete(id: Int): Int
```

**Parameters:**

- `id`: Primary key of feedback record to delete

**Returns:**

- `Int`: Number of records deleted (0 or 1)

**Error Handling:**

- Throws `Exception` if database operation fails

**Use Cases:**

- Remove duplicate feedback entries
- Delete spam or inappropriate feedback
- Manage database storage

**Example:**

```kotlin
val deletedCount = repositorySuggestion.delete(feedbackId)
Log.d("Feedback", "Deleted $deletedCount feedback record(s)")
```

## Implementation Details

### Hilt Integration

The RepositorySuggestion is injected via Hilt:

```kotlin
class RepositorySuggestion @Inject constructor(
   private val daoSuggestion: DaoSuggestion
) : IRepositorySuggestion
```

**Dependencies:**

- `DaoSuggestion` - Database access object

### Logging

All operations logged with TAG = "RepositorySuggestion":

- **DEBUG**: Operation start, success, record count
- **ERROR**: Operation failures with exception details

### Coroutine Context

All database operations run on `Dispatchers.IO` for non-blocking database access.

## Error Handling

| Error Type                   | Cause                  | Response         | Recovery                    |
|------------------------------|------------------------|------------------|-----------------------------|
| Database constraint violated | Duplicate/invalid data | Exception thrown | Validate data before insert |
| Database operation timeout   | DB locked or slow      | Exception thrown | Retry with backoff          |
| Disk space full              | Storage exhausted      | Exception thrown | Cleanup or request storage  |

## Data Flow Example

### Complete Feedback Submission Flow

```kotlin
// 1. Create feedback object
val feedback = Suggestion(
   fullName = "Jane Smith",
   message = "Add push notifications for severe weather alerts",
   createdAtUtc = TimeUtil.getCurrentUtcDateTime()
)

// 2. Store to database via repository
try {
   val feedbackId = repositorySuggestion.insert(feedback)
    Log.d("Feedback", "Successfully stored feedback with ID: $feedbackId")

    // 3. Optional: Show confirmation to user
    showSuccessMessage("Thank you for your feedback!")

} catch (e: Exception) {
    Log.e("Feedback", "Failed to store feedback: ${e.message}")
    showErrorMessage("Could not save feedback. Please try again.")
}
```
