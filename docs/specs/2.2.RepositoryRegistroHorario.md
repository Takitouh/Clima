# 2.2 Repository RegistroHorario - Hourly Weather Data

## Overview

Implementation of the hourly weather repository layer. Handles hourly temperature records that
relate to daily weather records via 1-to-Many relationship. Provides current weather fetching with
smart upsert logic (only updates if temperatures differ).

**Location:** `data/repository/impl/RepositoryRegistroHorario.kt`

**Interface:** `data/repository/IRepositoryRegistroHorario.kt`

**Status:** ✅ COMPLETE - Implementation finished with current weather support

## Entity: RegistroHorario (Hourly Weather)

```kotlin
@Entity(
    tableName = "registro_horario",
    foreignKeys = [
        ForeignKey(
            entity = RegistroClima::class,
            parentColumns = ["id"],
            childColumns = ["registro_clima_id"],
            onDelete = ForeignKey.CASCADE
        )
    ]
)
data class RegistroHorario(
    @PrimaryKey(autoGenerate = true)
    var id: Int = 0,
    @ColumnInfo(name = "registro_clima_id")
    var registroClimaId: Int,         // FK → RegistroClima
    var hour: Int,                    // 0-23 (hour of day)
    @ColumnInfo(name = "temp_fahrenheit")
    var tempFahrenheit: Float,        // Hourly temperature
    @ColumnInfo(name = "temp_celsius")
    var tempCelsius: Float            // Hourly temperature
)
```

**Key Features:**

- ✅ No redundant `fecha` field (already in parent RegistroClima)
- ✅ No redundant `ubicacion` field (already in parent RegistroClima)
- ✅ Foreign Key with CASCADE delete (parent delete → child deletes)
- ✅ Separated temperatures: Fahrenheit and Celsius
- ✅ Hour field: 0-23 (24-hour format)

## Relationship: RegistroClima ↔ RegistroHorario

### One-to-Many Pattern

```
RegistroClima (Daily Weather)
│ 1
├─ date: "2026-04-26"
├─ city: "New York"
├─ region: "NY"
├─ country: "United States"
├─ maxTempFahrenheit: 72.0
├─ minTempFahrenheit: 56.0
└─ maxTempCelsius: 22.2
   minTempCelsius: 13.3
   │
   │ Many
   ├─→ RegistroHorario (Hour 10)
   │   ├─ id: 1
   │   ├─ registroClimaId: 1 (FK)
   │   ├─ hour: 10
   │   ├─ tempFahrenheit: 65.0
   │   └─ tempCelsius: 18.3
   │
   ├─→ RegistroHorario (Hour 14)
   │   ├─ id: 2
   │   ├─ registroClimaId: 1 (FK)
   │   ├─ hour: 14
   │   ├─ tempFahrenheit: 70.0
   │   └─ tempCelsius: 21.1
   │
   └─→ RegistroHorario (Hour 18)
       ├─ id: 3
       ├─ registroClimaId: 1 (FK)
       ├─ hour: 18
       ├─ tempFahrenheit: 62.0
       └─ tempCelsius: 16.7
```

### Cascade Delete Behavior

```
DELETE FROM registro_clima WHERE id = 1
    ↓
Foreign Key CASCADE delete triggered
    ↓
DELETE FROM registro_horario WHERE registro_clima_id = 1
    ↓
All hourly records for that day automatically deleted ✅
```

## Repository Interface: IRepositoryRegistroHorario

### 0. fetchAndSaveCurrentWeather() [PRIMARY ENTRY POINT]

**Fetches current weather data and stores/updates hourly record for today with smart upsert logic.**

```kotlin
suspend fun fetchAndSaveCurrentWeather(locationInfo: LocationInfo): RegistroHorario?
```

**Parameters:**

- `locationInfo`: Location information containing city, region, country, latitude, longitude

**Returns:**

- `RegistroHorario` with current hour and temperatures from API if successful
- `null` if API fails or parent record cannot be created

**Process:**

1. Fetches or creates today's `RegistroClima` record (uses UTC date)
2. Calls WeatherAPI `current.json` endpoint with lat/lon coordinates
3. Extracts hour from API response's `location.localtime` (0-23)
4. Checks if hourly record exists for this hour
    - If exists & temperatures differ → **UPDATE** record
    - If exists & temperatures same → **no action** (skip unnecessary write)
    - If not exists → **INSERT** new record
5. Returns stored `RegistroHorario` with ID

**Key Features:**

- ✅ Smart upsert: only updates if temps differ (saves DB writes)
- ✅ UTC date storage: ensures consistency across timezones
- ✅ Automatic parent: creates daily record if needed
- ✅ Lat/lon query: uses coordinates for precise location matching
- ✅ Hour validation: ensures 0-23 range

**Error Handling:**

- Returns `null` if location info is invalid
- Returns `null` if today's daily record cannot be created or retrieved
- Throws exception if API request fails
- Throws exception if database operation fails
- Throws `IllegalArgumentException` if extracted hour is outside 0-23 range

**Example:**

```kotlin
val locationInfo = LocationInfo(
    city = "New York",
    region = "NY",
    country = "United States",
    latitude = 40.7128,
    longitude = -74.0060
)

val hourly = repository.fetchAndSaveCurrentWeather(locationInfo)
if (hourly != null) {
    println("Hour ${hourly.hour}: ${hourly.tempCelsius}°C")
    // Output: Hour 14: 22.5°C
} else {
    println("Could not fetch current weather")
}
```

## Data Flow Diagram

### Current Weather Flow (PRIMARY USE CASE)

```
LocationInfo (from LocationUtil)
        │
        ▼
RepositoryRegistroHorario.fetchAndSaveCurrentWeather()
        │
        ├─→ Call RepositoryRegistroClima.obtainOrFetchHistoricalWeather()
        │   ├─ Try: Query today's record from DB
        │   └─ Fallback: Fetch historical data from API & save
        │
        ├─→ Resolve RegistroClima ID
        │
        ├─→ Call WeatherApiService.getCurrentWeather()
        │   ├─ Query: "lat,lon" (e.g., "40.7128,-74.0060")
        │   ├─ Endpoint: v1/current.json
        │   └─ Response: Current temps + location.localtime
        │
        ├─→ Extract Hour from location.localtime
        │   └─ Format: "YYYY-MM-DD HH:mm" → Hour (0-23)
        │
        ├─→ Check if RegistroHorario exists for (registroClimaId, hour)
        │   │
        │   ├─→ If EXISTS:
        │   │   ├─ Compare temperatures
        │   │   ├─ If CHANGED → UPDATE record
        │   │   └─ If SAME → skip (no DB write)
        │   │
        │   └─→ If NOT EXISTS:
        │       └─ INSERT new record
        │
        └─→ Return RegistroHorario (with ID)
```

### Historical Weather Flow (USED BY OTHER LAYERS)

```
WeatherAPI Historical Response (includes hourly forecast data)
        │
        ▼
RepositoryRegistroClima.fetchHistoricalWeatherFromApi()
        │
        ├─→ Create RegistroClima (daily)
        │   │
        │   ├─→ Save to DB
        │   │
        │   └─→ Get registroClimaId
        │       │
        │       ▼
        │   For each hour in API forecast response:
        │       │
        │       ├─→ Extract RecordoHorario from forecast
        │       │
        │       └─→ RepositoryRegistroHorario.insertarRegistroHorario()
        │           │
        │           └─→ Save hourly record with FK to parent
        │
        └─→ Return WeatherRecordDto to caller
```

## Performance Considerations

### Query Optimization

- `obtenerPorRegistroClimaId()`: Uses indexed FK lookup `O(log n)`
- `obtenerPorRegistroClimaIdYHora()`: Composite query, indexed
- `obtenerTodos()`: Full table scan, avoid for large datasets

### Storage

- One hourly record per hour per day
- 24 records maximum per daily record
- Device hours = 24 × 365 = ~8,760 records per year per location

### Deletion

- Cascade delete ensures no orphaned records
- Deleting daily record = 24 hourly records auto-deleted

## Integration with RepositoryRegistroClima

The two repositories work together:

1. **RepositoryRegistroClima** creates daily record & provides today's ID
2. **RepositoryRegistroHorario** stores hourly details linked to daily record
    - Via `fetchAndSaveCurrentWeather()`: Fetches current weather & auto-links to today
    - Via CRUD methods: Direct record management
3. Called from **ViewModel** which orchestrates location retrieval and weather display

### Key Algorithms

#### Smart Upsert Logic

```kotlin
// Check if record exists for this hour
val existing = daoRegistroHorario.obtenerPorRegistroClimaIdYHora(
    registroClimaId,
    currentHour
)

// Compare temperatures
val tempsChanged =
    (existing.tempFahrenheit != apiTempFahrenheit) ||
            (existing.tempCelsius != apiTempCelsius)

// Only update if changed
if (tempsChanged) {
    daoRegistroHorario.actualizar(updated)  // UPDATE
} else {
    // Skip - no DB write
}
```

#### Hour Extraction from Localtime

```kotlin
// Input: "2026-04-27 14:30"
val hour = localtime.split(" ")[1]      // "14:30"
    .split(":")[0]                       // "14"
    .toInt()                             // 14
```
